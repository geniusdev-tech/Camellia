# Sentinel Security & Architecture Log

## 2026-04-03: Render Deployment & CI Fix

### Findings
- Render deployment failing due to incorrect host binding (127.0.0.1 instead of 0.0.0.0).
- Static frontend (Next.js) was not being served by the Flask backend.
- CI `bench` job failing because `scripts/argon2_bench.py` was missing from the repository.

### Actions
- Updated `app.py` to serve `static/dist` as the root and added catch-all routing for SPA.
- Modified `render.yaml` to use `env: docker` and improved environment variable security.
- Re-implemented `scripts/argon2_bench.py` to restore CI functionality.
- Fixed host binding logic in `app.py` for production environments.
- Ensured local `flask_session/` artifacts are excluded from the repository.

### Security Note
- `SECRET_KEY` in production is now generated by Render's `generateValue: true`.
- Session cookies are hardened with `Secure`, `HttpOnly`, and `SameSite=Strict` in production.
