# Sentinel Security & Architecture Log

## 2026-04-03: Render Deployment Resolution

### Findings
- The application was failing on Render due to incorrect host binding (127.0.0.1 instead of 0.0.0.0) and lack of frontend static serving in the Flask backend.
- The repository uses a multi-stage Dockerfile which is more reliable for Render than the native Python environment when both Node.js and Python are required.

### Actions
- Updated `app.py` to serve `static/dist` as the root for a Next.js static export.
- Implemented a catch-all route in Flask to support client-side routing (SPA).
- Updated `render.yaml` to use `env: docker` and improved environment variable security.
- Fixed host binding logic in `app.py` to use `0.0.0.0` in production.
- Removed local `flask_session/` artifacts to prevent security risks.

### Security Note
- `SECRET_KEY` is now generated by Render's `generateValue: true` to ensure a unique, strong key per deployment.
- CSRF protection (SeaSurf) is active in production for sensitive blueprints.
